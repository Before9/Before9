docker打包镜像

发表于 2018-05-20 分类于 部署

在docker hub上有时我们找不到想要的image,或自己想要打包纯净的image ( 怕投毒?? ).

所以自己制作docker image是一种不错的选择

拉取镜像

docker pull centos

查看镜像

docker images

创建容器

docker run -it –name=容器名 镜像名 /bin/bash

启动容器

docker start 容器ID

进入容器

docker exec -itd 容器名(容器ID) /bin/bash
docker attach 容器ID

安装ssh

yum install openssh-server

没有正常启动的话通常需要生成当前主机的ssh-key

ssh-keygen -t dsa -f /etc/ssh/ssh_host_dsa_key
ssh-keygen -t rsa -f /etc/ssh/ssh_host_rsa_key
ssh-keygen -t ecdsa -f /etc/ssh/ssh_host_ecdsa_key
ssh-keygen -t dsa -f /etc/ssh/ssh_host_ed25519_key

在容器中启动ssh服务，让外界可以连接

/usr/sbin/sshd

修改容器内部密码

修改密码需要安装passwd
yum install passwd.x86_64
修改密码
passwd root

退出容器

exit

修改云主机密码

修改云主机密码需要退出容器在执行passwd

查看所有容器

docker ps -a

制作镜像

docker commit -a “作者” -m “简介” 被打包的容器ID 镜像名: 版本号

镜像导出

docker save 镜像名 > ./文件名
例如： docker save ssh_image > ./文件名

加载镜像(解压)

docker load < ./文件名
例如： docker load < ./文件名

查看镜像是否存在

docker images

创建容器并映射端口把镜像跑起来

docker run -itd -p 11111:22(映射的端口号) 镜像ID /bin/bash

# docker
